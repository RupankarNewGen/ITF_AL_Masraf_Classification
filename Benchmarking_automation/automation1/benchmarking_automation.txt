====================================================
Benchmarking Pipeline Automation Documentation
====================================================

This documentation explains how to execute the end-to-end automated benchmarking pipeline.

-----------------------------
1. Environment Requirements
-----------------------------
Both automation scripts require the following Python environment to ensure dependencies run correctly:
Environment Path: /datadrive2/IDF_AL_MASRAF/qwen_env

You can activate it using:
source /datadrive2/IDF_AL_MASRAF/qwen_env/bin/activate
Or call the Python executable directly using:
/datadrive2/IDF_AL_MASRAF/qwen_env/bin/python <script_name>

-----------------------------
2. Phase 1: Data Preparation
-----------------------------
Script Name: automate_phase_1.py
Location: /datadrive2/IDF_AL_MASRAF/ITF_Al_Masraf_Classification/automate_phase_1.py

What it does:
1. Opens `FINAL_MATCHED_DIRECTORIES.json`.
2. Loops through every document directory defined in that JSON file.
3. Automatically executes `get_images_from_annoation_folders.py` for each directory.
4. Categorizes the images into distinct classes based on the predefined mapping.

How to run:
/datadrive2/IDF_AL_MASRAF/qwen_env/bin/python automate_phase_1.py

Where the results go:
The processed images are organized and saved into:
/datadrive2/IDF_AL_MASRAF/ITF_Al_Masraf_Classification/Data/Raw_ouput_data/images_for_inference

-----------------------------
3. Phase 2: Inference & Extraction Evaluation
-----------------------------
Script Name: automate_phase_2.py
Location: /datadrive2/IDF_AL_MASRAF/ITF_Al_Masraf_Classification/automate_phase_2.py

What it does:
1. Runs `idp_full_pipeline_api_request_nested.py` to hit the API with all categorised images from Phase 1. (Images categorized as 'Others' are ignored for inference accuracy calculation).
2. Parses the generated inference reports. If a document scores >= 80% inference accuracy, it proceeds to extraction benchmarking.
3. Passes qualified documents to `benchmarking_data_constructor.py` logic to create organized destination folders.
4. Executes `yolo_to_gt_converter.py` for each document class to format actual Ground Truth (GT) `.txt` annotations.
5. Invokes `report_generator_rupankar_v3.py` for each document class to generate extracted metrics against Ground Truths.
6. Parses the metric report `.txt` files generated by the report tool and extracts the "Complete_Match_Percentage" per class.
7. Condenses all overall inferences and granular extracted metrics into a master dictionary per document folder and saves it.

Important Configuration (Classes Mapping):
Before running Phase 2, if you need to map specific `classes.txt` paths for processing the Ground Truth annotations, open `automate_phase_2.py` and modify the `CLASSES_FILE_MAP` dictionary at the top of the file to point to your specific Paths.

How to run:
/datadrive2/IDF_AL_MASRAF/qwen_env/bin/python automate_phase_2.py

Where the results go:
The final aggregated extraction accuracy mapping for every document and their internal classes is saved at:
/datadrive2/IDF_AL_MASRAF/ITF_Al_Masraf_Classification/model_evaluation_codebase/extrction_benchmarking_codebase/Working_Data/benchmarking_final_results.json

All intermediate working labels and evaluation generated CSV/TXTs reside at:
/datadrive2/IDF_AL_MASRAF/ITF_Al_Masraf_Classification/model_evaluation_codebase/extrction_benchmarking_codebase/Working_Data/<Document_Directory>/...
